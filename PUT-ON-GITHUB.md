# 📤 How to put the Chai Quiz on GitHub (step by step)

Two situations below. Pick the one that matches you.

---

## A) FIRST TIME — making the website

Do this once.

1. Go to **https://github.com/new**
2. **Repository name:** type `chai-quiz`
3. Choose **Public**
4. Tick **Add a README file**
5. Click the green **Create repository** button
6. On the new page, click **Add file** (top right) → **Upload files**
7. Open the `deploy` folder on your computer. Select **ALL the files inside it**
   (click one, then press **Ctrl+A** to select them all).
8. **Drag them** onto the GitHub upload box.
   ⚠️ Drag the *files themselves*, NOT the `deploy` folder around them.
9. Wait for them to finish uploading (you'll see a list of file names).
10. Scroll down → click the green **Commit changes** button.

### Now turn the website ON:
11. Click the **Settings** tab (top of the repo).
12. On the left, click **Pages**.
13. Under **Source**, pick **Deploy from a branch**.
14. **Branch:** choose **main**, folder **/ (root)**, then click **Save**.
15. Wait about 1 minute. Your website link appears at the top of that Pages screen:
    **https://aayansh.github.io/chai-quiz**

Done! Share that link with the class. 🎉

---

## B) UPDATING — when I give you new files

Do this every time I say "upload these files".

1. Go to your repo: **https://github.com/aayansh/chai-quiz**
2. Click **Add file** (top right) → **Upload files**
3. Drag in the new versions of the files I named.
4. When it asks, let it **replace / overwrite** the old ones.
5. Scroll down → click **Commit changes**.
6. Wait ~30 seconds.
7. Open your website and press **Ctrl + Shift + R** (this forces a fresh load).

That's it — the new version is live.

---

## 💡 Tips

- You can upload **just the files that changed** — you don't have to re-upload everything.
- If something looks old after updating, it's almost always the browser cache:
  press **Ctrl + Shift + R** to hard-refresh.
- File names must stay **exactly the same** (don't rename them), or the game won't find them.

---

## Which files do I upload?

- **First time:** ALL files from the `deploy` folder.
- **Updating:** only the ones I list for that change (I'll always tell you).
